Élection Présidentielle Française 2017

29 April, 2017

Basic Setup

I had used R in the past to collect tweets, but this technique has its limitations, when one wants to collect a lot of data and do so 24/7. In my opinion there is no way around Python in this case and in the best case a server is at hand as well. It is the first time I am using this combination, I did some research to find a setup that is fast, since I was expecting a lot of tweets around the election days. My setup is based on this great twitter scraper repo. In the first days I only had a few words to track specified, but adjusted my list based on a simple preliminary analysis of collected data.

In the following sub pages I present some basic analysis of collected tweets in the light of the presidential elections in France.

Track terms

these are the terms for which Twitter’s API was looking in tweets to forward matches.

  • “presidentielle2017”, “presidentielles2017”, “presidentielle”, “presidentielles”, “#France2017”, “#Frenchelections”
  • “Fillon”, “Fillon2017”, “projetfillon”, “jevotefillon”, “fillongate”,”francoisfillon”
  • “Hamot”, “Hamot2017”,
  • “LePen”, “Le Pen”, “marielepen”, “marine2017”, “#mlp”,
  • “Macron”, “enmarche”, “macron2017”, “jevotemacron”, “emmanuelmacron”
  • “Melenchon”, “jlm2017”, “jlmelenchon”,
  • “Arthaud”,
  • “Poutou”,
  • “Asselineau”, “Asselineau2017”,
  • “Dupont-Aignan”, “DupontAignan”,
  • “Lassalle”,
  • “Cheminade”

In total my dataset contains 8,783,948 tweets.
